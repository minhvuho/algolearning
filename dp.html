1. Giới thiệu

Quy hoạch động (Dynamic Programming - DP) là một kỹ thuật giải quyết bài toán bằng cách chia bài toán lớn thành nhiều bài toán con nhỏ hơn.

Kết quả của các bài toán con sẽ được lưu lại để tránh tính toán lại nhiều lần.

DP thường được sử dụng khi bài toán có hai tính chất quan trọng:

  • Optimal Substructure: nghiệm tối ưu của bài toán có thể xây dựng từ nghiệm tối ưu của các bài toán con.
  • Overlapping Subproblems: các bài toán con lặp lại nhiều lần.

2. Ý tưởng của quy hoạch động

Thay vì giải lại cùng một bài toán con nhiều lần, DP lưu kết quả của nó vào một bảng (thường là mảng hoặc vector).

Khi cần sử dụng lại kết quả, chương trình chỉ cần đọc lại giá trị đã lưu.

Có hai cách triển khai phổ biến:

  • Top-down (Memoization): dùng đệ quy kết hợp lưu kết quả.
  • Bottom-up: xây dựng bảng DP từ các trạng thái nhỏ đến lớn.

3. Các bước xây dựng thuật toán DP

Thông thường khi giải một bài toán DP ta làm theo các bước:

  • Xác định trạng thái (state)
  • Tìm công thức chuyển trạng thái (transition)
  • Xác định giá trị khởi tạo (base case)
  • Tính toán các trạng thái theo thứ tự phù hợp

4. Ví dụ: bài toán Fibonacci

Bài toán: tính số Fibonacci thứ n.

Công thức: 


F(n) = F(n-1) + F(n-2)

Giá trị khởi tạo: 


F(0) = 0
F(1) = 1

Nếu dùng đệ quy đơn thuần, chương trình sẽ tính lại rất nhiều lần.

DP giúp giảm đáng kể số phép tính.

5. Code C++

Ví dụ tính Fibonacci bằng phương pháp Bottom-up: 


#include <bits/stdc++.h>
using namespace std;

int main(){

int n;
cin >> n;

vector<long long> dp(n+1);

dp[0] = 0;
dp[1] = 1;

for(int i=2;i<=n;i++)
dp[i] = dp[i-1] + dp[i-2];

cout << dp[n];

}

6. Độ phức tạp

Với DP, mỗi trạng thái chỉ được tính một lần.

Độ phức tạp thường là: 


O(số trạng thái × số chuyển trạng thái)

7. Một số bài toán áp dụng

  • Fibonacci
  • Knapsack (ba lô)
  • Longest Increasing Subsequence
  • Longest Common Subsequence
  • Coin Change
  • Matrix Chain Multiplication

Quy hoạch động là một trong những kỹ thuật quan trọng nhất trong competitive programming.
