1. Giới thiệu

DP Bitmask là một kỹ thuật quy hoạch động dùng để giải các bài toán có số lượng trạng thái nhỏ nhưng cần biểu diễn tập hợp các phần tử.

Thay vì dùng mảng hoặc vector để lưu tập hợp, ta dùng **bitmask** để biểu diễn trạng thái.

Kỹ thuật này thường được dùng trong các bài toán như:

  • Travelling Salesman Problem (TSP)
  • Ghép cặp
  • Phân công công việc
  • Hamiltonian path

2. Bitmask là gì

Bitmask là cách biểu diễn một tập hợp bằng các bit của một số nguyên.

Ví dụ: 


mask = 13

Binary: 1101

Tập hợp tương ứng: 


{0,2,3}

Các phép toán thường dùng: 


(mask & (1<<i))      // kiểm tra phần tử i có trong tập không

mask | (1<<i)       // thêm phần tử i

mask ^ (1<<i)       // đảo bit

mask & ~(1<<i)      // xóa phần tử i

3. Trạng thái DP

Trong DP Bitmask, trạng thái thường có dạng: 


dp[mask]

hoặc: 


dp[mask][i]

Trong đó:

  • mask biểu diễn tập hợp các phần tử đã chọn
  • i là vị trí hiện tại

DP sẽ chuyển trạng thái bằng cách thêm hoặc bỏ một phần tử khỏi mask.

4. Ví dụ bài toán

Travelling Salesman Problem (TSP):

Cho n thành phố và ma trận khoảng cách giữa các thành phố.

Hãy tìm đường đi ngắn nhất bắt đầu từ thành phố 0, đi qua tất cả các thành phố đúng một lần và quay lại điểm xuất phát.

Trạng thái DP: 


dp[mask][i]

Ý nghĩa:

  • mask: các thành phố đã đi qua
  • i: thành phố hiện tại

5. Code C++


#include <bits/stdc++.h>
using namespace std;

const int INF = 1e9;

int dist[20][20];
int dp[1<<15][15];

int main(){

int n;
cin >> n;

for(int i=0;i<n;i++)
for(int j=0;j<n;j++)
cin >> dist[i][j];

for(int mask=0;mask<(1<<n);mask++)
for(int i=0;i<n;i++)
dp[mask][i] = INF;

dp[1][0] = 0;

for(int mask=1;mask<(1<<n);mask++){

for(int u=0;u<n;u++)
if(mask & (1<<u)){

for(int v=0;v<n;v++)
if(!(mask & (1<<v))){

int newmask = mask | (1<<v);

dp[newmask][v] = min(
dp[newmask][v],
dp[mask][u] + dist[u][v]
);

}

}

}

int ans = INF;

for(int i=0;i<n;i++)
ans = min(ans, dp[(1<<n)-1][i] + dist[i][0]);

cout << ans;

}

6. Độ phức tạp

Số trạng thái của mask là: 


2^n

Nếu mỗi trạng thái có n chuyển tiếp: 


O(n^2 * 2^n)

Do đó kỹ thuật này thường chỉ áp dụng khi: 


n ≤ 20
